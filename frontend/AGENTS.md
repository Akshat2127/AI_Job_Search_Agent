# Frontend Instructions

Follow the root `AGENTS.md`. Build an accessible React + TypeScript client with strict types and a centralized API client. Every data view needs useful loading, empty, error, and retry states. Forms require client validation while treating server validation as authoritative.

Never imply that an application, email, calendar event, or AI result is final without an explicit user action and visible state. Clearly label deterministic results, AI recommendations, extracted facts, and user-approved content. Avoid rendering untrusted HTML; external links must be safe and visibly identifiable.

Frontend changes require component tests, TypeScript, lint, and a production build. Keyboard navigation and meaningful labels are acceptance requirements, not optional polish.
